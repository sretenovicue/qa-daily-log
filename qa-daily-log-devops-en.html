2. System Architecture (v2.0)
┌──────────────────────────────────────────────────────────────────────────┐ │ BROWSER │ │ https://qa-daily-log.online │ └──────────────────────────────┬───────────────────────────────────────────┘ │ HTTPS port 443 ▼ ┌───────────────────────────────────────────────────────────────────────┐ │ CLOUDFLARE EDGE (free plan) │ │ • SSL termination (HTTPS ↔ Tunnel) │ │ • DDoS protection — Pi's real IP is NEVER exposed to the internet │ │ • DNS proxy (CNAME Tunnel record) │ └───────────────────────────────┬───────────────────────────────────────┘ │ Cloudflare Tunnel (TLS, outbound connection — ISP port blocking irrelevant) ▼ ┌───────────────────────────────────────────────────────────────────────┐ │ RASPBERRY PI 4B (hostname: nasty, Debian Linux) │ │ │ │ cloudflared (systemd) ──────→ http://localhost:3001 │ │ │ │ │ PM2 → node server.js (port 3001) │ │ │ │ │ │ │ ├── /api/* → Express routes │ │ │ └── /* → React dist/ ───┘ │ └───────────────────────────────┬───────────────────────────────────────┘ │ PostgreSQL SSL (Neon cloud, Frankfurt) ▼ ┌───────────────────────────────────────────────────────────────────────┐ │ NEON POSTGRESQL (serverless, free tier) │ │ ep-jolly-king-a9bnwjuq — gwc.azure.neon.tech │ └───────────────────────────────────────────────────────────────────────┘
Key advantage of Cloudflare Tunnel: The Pi requires no open inbound ports. Cloudflared establishes an outbound connection to Cloudflare — ISP blocking of inbound ports (80, 443) has no effect. The Pi's real IP remains hidden.

DNS Flow

2. DNS resolves to Cloudflare nameservers
3. Cloudflare proxy receives the request
4. Cloudflare forwards request through Tunnel
5. cloudflared on Pi receives and forwards to :3001
6. Express responds with HTML or JSON

Useful SQL queries (Neon console):
-- Set manager role UPDATE users SET role = 'manager' WHERE email = 'email@domain.com'; -- Deactivate / activate a user UPDATE users SET active = false WHERE username = 'user'; UPDATE users SET active = true WHERE username = 'user'; -- Approve user (approved = can log in) UPDATE users SET approved = true WHERE email = 'email@domain.com'; -- View all users SELECT id, email, username, role, active, approved FROM users;
