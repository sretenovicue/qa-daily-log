2. Arhitektura sistema
┌─────────────────────────────────────────────────────────────────┐ │ BROWSER │ └─────────────────────┬───────────────────────────────────────────┘ │ HTTPS ▼ ┌─────────────────────────────────────┐ │ NETLIFY CDN (frontend) │ │ React SPA — qa-daily-log.netlify │ └──────────┬──────────────────────────┘ │ /api/* → proxy redirect (netlify.toml) ▼ ┌─────────────────────────────────────┐ │ RENDER.COM (backend) │ │ Node.js + Express — port auto │ └──────────┬──────────────────────────┘ │ PostgreSQL (SSL, rejectUnauthorized) ▼ ┌─────────────────────────────────────┐ │ NEON POSTGRESQL (cloud DB) │ │ Frankfurt — serverless Postgres │ └─────────────────────────────────────┘
Ključna napomena: Browser nikad ne komunicira direktno sa backendom. Sve /api/* rute prolaze kroz Netlify proxy — backend URL ostaje skriven, CORS je centralizovan.

Detalji

ServisNeon.tech (free, serverless)
RegionFrankfurt (Azure Germany)
Projektqatrackapp
Branchproduction
Databaseneondb

Šema tabela

users id, email, username, password_hash role: 'user' | 'manager' | 'guest' active: boolean approved: boolean title: text (nullable) avatar_data: text (base64, nullable) created_at entries id, task_number, category, action status, description, duration project, date, time, created_at user_id → users(id) CASCADE
Korisni SQL upiti:
-- Postavi manager rolu UPDATE users SET role = 'manager' WHERE email = 'email@domen.com'; -- Deaktiviraj korisnika (ne može da se loguje) UPDATE users SET active = false WHERE username = 'korisnik'; -- Reaktiviraj korisnika UPDATE users SET active = true WHERE username = 'korisnik'; -- Pregled svih korisnika SELECT id, email, username, role, active FROM users;

8. Sigurnost
MjeraImplementacija
HTTP security headershelmet.js — CSP, HSTS, X-Frame-Options, X-Content-Type-Options...
Brute force zaštitaexpress-rate-limit — max 10 pokušaja / 15 min / IP na login i register
Hashovanje lozinkibcryptjs — salt rounds: 10
AutentifikacijaJWT token (7 dana), payload: { userId, role }
CORSSamo ALLOWED_ORIGIN domen (Netlify URL)
Payload limit50 KB maksimalni request body
SSL prema DBrejectUnauthorized: true u produkciji
Tajni podaci.env u .gitignore — nikad u GitHub-u
