11. Baza — Neon PostgreSQL

Detalji

ServisNeon.tech (free, serverless)
RegionFrankfurt (Azure Germany West)
Projektqatrackapp
Branchproduction
Databaseneondb

Šema tabela

users id, email, username, password_hash role: 'user' | 'manager' | 'guest' active: boolean approved: boolean title: text (nullable) avatar_data: text (base64, nullable) created_at entries id, task_number, category, action status, description, duration project, date, time, created_at user_id → users(id) CASCADE
Korisni SQL upiti (Neon konzola):
-- Postavi manager rolu UPDATE users SET role = 'manager' WHERE email = 'email@domen.com'; -- Deaktiviraj / aktiviraj korisnika UPDATE users SET active = false WHERE username = 'korisnik'; UPDATE users SET active = true WHERE username = 'korisnik'; -- Odobri korisnika (approved = može se prijaviti) UPDATE users SET approved = true WHERE email = 'email@domen.com'; -- Pregled svih korisnika SELECT id, email, username, role, active, approved FROM users;

MjeraImplementacija
HTTP security headershelmet.js — CSP, HSTS, X-Frame-Options, X-Content-Type-Options
Brute force zaštitaexpress-rate-limit — max 10 pokušaja / 15 min / IP (login + register)
Hashovanje lozinkibcryptjs — salt rounds: 10
AutentifikacijaJWT token (7 dana), payload: { userId, role }
CORSSamo ALLOWED_ORIGIN=https://qa-daily-log.online
Payload limit50 KB maksimalni request body
SSL prema DBsslmode=require&channel_binding=require u Neon URL-u
SQL injectionParametrizovani upiti svuda — nema string concatenation
